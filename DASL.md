# Dynamic Affordance Sync Ledger (DASL)

## Computational Shared Mental Model (SMM)

**1. Agent Hierarchy:**
*   **Orchestrator (Human/User):** Provides seed concepts and triggers generation. Operates within the deterministic UI space.
*   **Mediator (Angular UI):** Parses user intent, selects personas, and structures the request. Translates fluid LLM responses into rigid visual affordances (cards).
*   **Oracle (Gemini LLM):** Generates content based on Epistemic Engineer Persona and PARADOX constraints. Operates in a highly dimensional, fluid semantic space.

**2. Epistemic Assumptions:**
*   **Assumption 1 (The UI Constraint):** The system assumes that all complex, paraconsistent logic generated by the Oracle can be effectively flattened into a "Title + Paragraph" binary structure (via `parseStandardResponse`).
*   **Assumption 2 (The Node.js Fallacy):** The system assumes a traditional server environment (`process.env.API_KEY`), necessitating a browser hack to maintain functionality in a pure frontend context.

**3. Causality & Information Flow:**
*   User Input -> Angular Component -> `gemini.service.ts` -> Gemini API.
*   Gemini API -> `gemini.service.ts` (`parseStandardResponse`) -> Angular Component -> Konva.js Canvas.

## Semantic Parallax Zones (SPZs)

*   **SPZ-Alpha (The Formatting Paradox):** [RESOLVED] The prompt demands the preservation of contradictions and avoidance of "Ontological Flattening," yet the previous code explicitly split the response by `**` tokens. This has been resolved by replacing the naive string split with a non-destructive regex parser that correctly handles nested markdown elements without destroying the intended structure.
*   **SPZ-Beta (The Environment Ghost):** The `gemini.service.ts` relies on `process.env`, a Node.js construct, in an Angular browser environment. The patch (`patch_process.py`) creates a phantom dimension where `process` exists just enough to avoid a crash, but not enough to provide true security or configuration management.

**4. Future State & Forward-Thinking Mechanics (Post-Planning Phase):**
*   **MADS (Multi-Agent Dialectical Synthesis):** The system must evolve to handle multi-persona state simultaneously. The assumption that the UI can only hold one persona context at a time must be broken to allow for synthetic generation (Thesis + Antithesis = Synthesis). This creates a new semantic zone where the LLM is expected to actively debate itself rather than just responding to the user.
*   **CLT (Conceptual Lineage Tracking):** The current conceptual model treats each generated output as an isolated node on the canvas. The future state requires a graph-based mental model where nodes have ontological parents, and visual lines represent semantic lineage, moving the canvas from a mere whiteboard to a temporal knowledge graph.
